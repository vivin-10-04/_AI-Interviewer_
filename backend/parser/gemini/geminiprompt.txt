Given the extracted text from a PDF resume, structure the information into a well-organized JSON format. The JSON should include key sections and fields commonly found in resumes, such as:



Contact Information: Name, email, phone number, location, city, pin code, date of birth, gender.

Personal Bio/ Description: A brief description of the applicant's background, interests, and career objectives.

Years of Experience: this should be calculated based on the dates given in workExperience add the years. if date is mentioned as '2023 - present' then calculate the years from 2023 till current year.

expectedSalary : Expected salary of the applicant. null if not available.

Prefered Location: Check for list of city locations given in the text. if not found, use the location field from contactInformation.

Professional Summary: A brief overview of the applicant's skills, experience, and career goals.

Work Experience: A list of previous employment positions, including company names, job titles, dates of employment, and key responsibilities. always list the work experience from oldest to latest as per the dates provided.

Education: A list of educational qualifications, including institutions attended, degrees obtained, and relevant coursework. always list the educational details from oldest to newest as per the dates provided.

Skills: A list of technical and soft skills relevant to the applicant's field.

Certifications: A list of professional certifications, including certification names, issuing organizations, issue dates, expiration dates, and links to certificates. keep this as null if not found.

Area of Interest: A list of the applicant's areas of interest or expertise.

Languages: A list of languages spoken by the applicant. keep as null if not found.

Prefered Job Role: A list of preferred job roles or positions sought by the applicant. keep null if not found

for any field with date, it should be in 'yyyy-mm-dd' or 'yyyy-mm' format only. in any case you should default to 1st day of year in case if month and date is missing, and default to 1st day of month in case if date is not mentioned. maintain this for all date fields without fail.

Ensure the JSON is properly formatted, with clear and consistent naming conventions for the fields and sections. The JSON should be optimized for easy readability and machine processing."

For any field which is not available in the text, keep it as null in the JSON.
